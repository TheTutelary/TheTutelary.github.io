01
Design Principles
Five directives that guide every decision in the system.
🏛️
App shell first
Persistent UI structure — header, sidebar, framing — should feel architectural, not decorative. Rely on spacing, alignment, and subtle contrast.
📐
Surface hierarchy over visual effects
Hierarchy comes from surface levels, border restraint, spacing, and typography — not blur, glow, or oversized rounded wrappers.
🎛️
Quiet controls
Buttons, inputs, and menu triggers should feel dependable and compact. They should not visually overpower content.
📦
Panels, not ornaments
Content containers are panels. They separate information cleanly without competing with navigation or controls for attention.
🔁
Consistency beats novelty
When choosing between a more expressive pattern and a more consistent one, always prefer consistency. Tribera grows through repeatable primitives, not one-off ideas.

12
Do / Don't
Anti-drift rules. When uncertainty arises, choose the calmer, more structural option.
✓ Do
✓ Use existing primitives from tribera-ui.css
✓ Follow the shell/panel/control separation strictly
✓ Use .Panel as the preferred semantic name for content containers
✓ Extend system semantically — fill reusable gaps only
✓ Use spacing and typography to create hierarchy before adding framing
✓ Keep badge content short — status or metadata
✗ Don't
✗ Introduce new arbitrary radius values
✗ Use pill treatment on headers or layout wrappers
✗ Add glass-heavy blur treatments
✗ Invent one-off card variants for isolated screens
✗ Use gradient as the default shell background
✗ Use .GlassCard in new work — it's a legacy alias
